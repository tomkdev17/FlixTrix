The name of this API is Harmonicks.

Users will be able to create/delete an account in which they'll be able to access information about music, such as songs, artist, and genres

Here are some examples of the types of HTTP requests you could make with their corresponding URL endpoints

Business Logic URL HTTP Request Body Format Response Body Format
Return all Songs /songs GET None A JSON object holding all songs
Return data about a single song by title /songs/:title GET None A JSON object holding all of the data concerning a single song (title, artist, genre, release)
Return a description of a genre by name (i.e. Jazz) /songs/genres/:genreName GET None A JSON object holding the description of a particular genre
Return information about an artist by name /songs/artists/:artistName GET None A JSON object holding data about a particular artist (artist bio, discography etc.)
Allow new users to register an account /users POST A JSON object containing the new User information Example:
`{"Name": "John Doe", "Favorite Songs": []}`		 A JSON object reflecting the new User input, and also including a user ID
Allow users to update their account info (username) /users/:id PUT A JSON object containing the updated parameters Example:
`{ "Username": "testuser", "Password": "testpassword123", "Email": "testuser@example.com", "Birthday": "1995-01-01", "Favorites": [] }`		 A JSON object reflecting the updated User data
Allow users to add a song to their favorites list /users/:id/:songTitle POST None A text message indicating that the song has been added to the favorites list
Allow users to remove a song from their favorites list /users/:id/:songTitle DELETE None A text message indicating that the song has been removed from the favorites list
Allow existing users to deregister their account /users/:id DELETE None A text message indicating that the user's account information has been removed
